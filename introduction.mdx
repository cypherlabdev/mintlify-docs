Welcome to the **AGG Aggregator Platform** API documentation.

AGG is a specialized sports data aggregation platform that unifies match data, live scores, statistics, and streaming URLs from multiple upstream providers into a single, cohesive interface.

## What Problems It Solves

Integrating with multiple sports data and streaming providers is complex. Providers have different API structures, authentication mechanisms, and rate limits. AGG simplifies this by providing a unified, standardized interface. You integrate once with AGG, and you get access to all underlying providers automatically.

## Why AGG vs Direct Integrations

| | AGG | Direct Provider Integrations |
|---|---|---|
| **Integration effort** | Single API, single auth | N providers, N APIs, N auth flows |
| **Data normalization** | Unified schema across all providers | Different schemas per provider |
| **Stream aggregation** | All stream URLs in one response | Separate calls per provider |
| **Real-time delivery** | WebSocket with unified event types | Varies: polling, webhooks, SSE |
| **Maintenance** | AGG handles upstream changes | You maintain each integration |

## Scope & Limitations

<Warning>
AGG only aggregates and distributes stream **URLs** — we do not host video content, transcode streams, or provide CDN services. Your application is responsible for video playback using the provided URLs.
</Warning>

* **Agnostic:** AGG abstracts away the specific upstream providers. You do not need to know which provider a stream or match comes from.
